General Notes

HEAD

  • The HEAD element contains information about the webpage
  • Information in the HEAD is not visible to the end user
  • Information within HEAD is for the web browser to read and display the webpage properly

BODY

  • The BODY element represents the visible content to the end user
  • HTML acts as the foundation of webpages

ELEMENTS

  • !DOCTYPE represents the document type, helping browsers correctly display the page
  • head contains information for the browser, this information is not visible to the user. it contains such things as the page title (seen in the tab), character set (usually "UTF-8" for English speaking countries and links to CSS Stylesheets, among other things
  • title defines the page's title
  • body contains all of the visible html on a page. There can ne only one body per page
  • header creates a page header
  • footer creates a page footer
  • h1 - h6 these elements create headings, with h1 being the most important and h6 being the least important Most browsers recognise headings and affect the text size accordingly
  • p creates a new paragraph
  • a creates a link. Links depend on having an href attribute inside of them with the link itself, for example: a href="url_destination"
  • img identifies an image, img elements must be paired with an src attribute, and should also contain an alt attribute. For example: img src="Location_of_Image" alt="Image_descriptio"
  • ul creates an unordered list
  • ol creates an ordered list
  • li creates list elements to be used inside ordered or unordered lists
  • section is used to contain multiple elements in one place, usually because of linking themes
  • div creates a division, similar to section
  • br creates a line break
  • aside defines content aside from page content
  • b affects text making it bold
  • i italicises text
  • link defines the relationship between an html document and an external resource (usually stylesheets)
  • nav defines a navigation bar
  • span defines a section within a document
