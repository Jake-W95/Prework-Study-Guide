Prework Study Guide

HTML

  • The HEAD element contains information about the webpage
  • Information in the HEAD is not visible to the end user
  • Information within HEAD is for the web browser to read and display the webpage properly
  • The BODY element represents the visible content to the end user
  • HTML acts as the foundation of webpages

CSS

  • A margin indicates how much space is around an elements perimeter
  • A padding indicates how much space is around the content INSIDE of the element
  • If using a seperate .css file, it must be linked to the HTML file using a "link" element
  • The "link" element will occur in the HEAD of the HTML, not the BODY
  • Colours can be indicated by written language (blue,orange etc) or by hexadecimal code (#888888)
  • Typing "." followed by a class (such as Card) will apply all CSS assignments to ALL HTML elements in that class

Git

  • ALL inputs relating to Git or Github must begin with "git"
  • "git log" shows the commit history of the currently active branch
  • "git status" shows modified files in working directory, staged for the next commit
  • "git branch" lists branches with an * appearing on the active branch
  • "git checkout" switches to another branch and checks the branch out into working directory
  • "git checkout -b new-branch-name-example" creates a new branch and switches to it
  • "git branch -d branch-name" will delete the named branch WILL NOT WORK IF IT IS THE CURRENT BRANCH

Bash Commands

  • "cd" (Change Directory) Navigates to directory held WITHIN current directory
  • "cd .." Navigates back one level through directory
